Was macht die App?

Die Anwendung dient zur einfachen Erfassung täglicher Lehrlings-Fahrten. Fahrer wählen Route, Punkte und Schicht, die Daten werden automatisch an den Server übertragen und gespeichert.

Bedienung

  • Route wählen: Oben zwischen Route 1 / 2 umschalten.
  • Punkte sortieren: Per Drag & Drop ordnen, aktivieren oder deaktivieren.
  • Gesamtstrecke: wird automatisch berechnet.
  • Speichern: Button Speichern sendet Daten an den Server.
  • Berichte: Unter Berichte sind die letzten Einträge abrufbar.
  • Admin: Unter Admin lassen sich Punkte, Matrix, Fahrer und Autos pflegen.
  • Darstellung: Hell/Dunkel-Modus über den Button 🌙 / ☀️.

Admin benutzen

  • Punkte: Hier lassen sich Haltepunkte anlegen, umbenennen, aktivieren/deaktivieren und in der Reihenfolge verschieben.
  • Matrix: Hier werden die Kilometer zwischen den Punkten gepflegt. Für neue Punkte müssen mindestens die realen Nachbar-Strecken eingetragen werden.
  • Fahrer: Fahrer-IDs, Namen und Aktiv-Status verwalten.
  • Autos: Kennzeichen und Fahrzeug-IDs verwalten.

Neue Haltestelle anlegen

  • In Admin zuerst unter Punkte die neue Haltestelle anlegen, z. B. 2R1a.
  • Danach unter Matrix die Strecken ergänzen, z. B. 2R1 → 2R1a und 2R1a → 2R2.
  • Wenn die Strecke auch rückwärts genutzt wird, auch die Gegenrichtung pflegen.
  • Erst danach auf Alles speichern klicken.

Wichtig

  • Die normale Fahrten-Seite rechnet nur sauber, wenn für die genutzten Punkt-Kombinationen Werte in der Matrix vorhanden sind.
  • Route 1 und Route 2 werden getrennt genutzt. Es müssen also nur die Strecken gepflegt werden, die im jeweiligen Ablauf wirklich vorkommen.
  • Nach Änderungen in der Admin-Seite die normale App kurz neu laden, damit die aktuellen Daten sichtbar sind.

Technik (Kurzinfo)

  • Frontend: HTML / CSS / Vanilla JS
  • Backend: Google Apps Script (GAS) mit JSON-API
  • Hauptdateien: api.js, main.js, function.js
  • Endpoints: ?fn=getdata, ?fn=recent, POST submit
  • Speicherung: Tabelle Submissions (Google Sheets)
